## SMART 目标设定

### S - Specific（具体的）

**核心问题**：谁？做什么？在哪里？为什么？

**目标描述**：
- 谁（负责人）：_______________________________
- 做什么（具体工作）：_______________________________
- 在哪里（地点/环境）：_______________________________
- 为什么（目的）：_______________________________

**检验问题**：
- [ ] 明确了负责人
- [ ] 描述了具体要完成的工作
- [ ] 说明了工作的具体内容
- [ ] 解释了为什么要做

---

### M - Measurable（可衡量的）

**核心问题**：如何量化？完成标准是什么？

**衡量指标**：

| 指标类型 | 具体指标 | 目标值 |
|----------|----------|--------|
| 数量指标 | | |
| 质量指标 | | |
| 时间指标 | | |
| 其他指标 | | |

**完成标准**：
- [ ] _______________________________
- [ ] _______________________________

**检验问题**：
- [ ] 有明确的完成标准
- [ ] 标准可量化或可验证
- [ ] 没有模糊的描述（如"尽量"、"大概"、"差不多"）

---

### A - Achievable（可实现的）

**核心问题**：有能力做吗？有资源吗？合理吗？

**可行性评估**：

| 评估项 | 评估结果 | 说明 |
|--------|----------|------|
| 团队能力 | □ 具备 □ 部分具备 □ 不具备 | |
| 时间资源 | □ 充足 □ 紧张 □ 不足 | |
| 技术条件 | □ 可行 □ 有风险 □ 不可行 | |
| 外部依赖 | □ 无依赖 □ 有依赖可控 □ 依赖风险高 | |
| 资源支持 | □ 已到位 □ 部分到位 □ 未到位 | |

**检验问题**：
- [ ] 负责人具备必要技能
- [ ] 有足够的时间和资源
- [ ] 目标具有挑战性但不过分
- [ ] 考虑了风险和不确定性

---

### R - Relevant（相关的）

**核心问题**：与大目标相关吗？值得做吗？优先级对吗？

**目标对齐**：

```
本工作包目标
     ↑ 支撑
上层任务目标：_______________________________
     ↑ 支撑
项目整体目标：_______________________________
```

**优先级评估**：

| 评估维度 | 权重 | 评分（1-5） | 加权分 |
|----------|------|------------|--------|
| 业务价值 | 30% | | |
| 紧急程度 | 25% | | |
| 依赖关系 | 20% | | |
| 资源可用性 | 15% | | |
| 风险程度 | 10% | | |
| **总分** | 100% | | |

**艾森豪威尔矩阵定位**：
- [ ] Q1 重要紧急 → 立即做
- [ ] Q2 重要不紧急 → 计划做
- [ ] Q3 不重要紧急 → 授权做
- [ ] Q4 不重要不紧急 → 减少做

**检验问题**：
- [ ] 支持上层任务的完成
- [ ] 对项目整体目标有贡献
- [ ] 优先级合理

---

### T - Time-bound（有时限的）

**核心问题**：截止日期是哪天？有里程碑吗？

**时间规划**：

| 时间节点 | 日期 | 交付物 |
|----------|------|--------|
| 开始时间 | | |
| 中期检查点 | | |
| 完成时间 | | |

**缓冲预留**：□ 已预留 ___% 缓冲时间

**检验问题**：
- [ ] 有具体的截止日期/时间
- [ ] 时间安排合理
- [ ] 考虑了前置依赖
- [ ] 设置了里程碑检查点
